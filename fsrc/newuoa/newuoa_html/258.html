5     ! Remarks:
6     !
7     ! 1. REAL*4, REAL*8, INTEGER*4, INTEGER*8 are not Fortran standard
8     !    expressions. Although they are supported by many compilers as
9     !    extensions, it is better to avoid them.
10     !
11     ! 2. Never use KIND with a literal value, e.g., REAL(KIND=8), because 
12     !    Fortran standards never define what KIND=8 means. There is NO 
13     !    guarantee that REAL(KIND=8) will be legal, let alone double precision.
14     !
15     ! 3. Fortran standard (as of F2003) specifies the following for types 
16     !    INTEGER and REAL. 
17     !
18     !    - A processor shall provide ONE OR MORE representation methods that 
19     !      define sets of values for data of type integer; if the kind type
20     !      parameter is not specified, the default kind value is KIND(0) and
21     !      the type specified is DEFAULT INTEGER. 
22     !    - A processor shall provide TWO OR MORE approximation methods that 
23     !      define sets of values for data of type real; if the type keyword 
24     !      REAL is specified and the kind type parameter is not specified, 
25     !      the default kind value is KIND (0.0) and the type specified is 
26     !      DEFAULT REAL; If the type keyword DOUBLE PRECISION is specified, 
27     !      the kind value is KIND (0.0D0) and the type specified is DOUBLE 
28     !      PRECISION real; the decimal precision of the double precision real
29     !      approximation method shall be greater than that of the default 
30     !      real method. 
31     !
32     !    In other words, the standard only imposes that the following three
33     !    types should be supported:
34     !    - INTEGER(KIND(0)), i.e., default integer,
35     !    - REAL(KIND(0.0)), i.e., default real (single-precision real),
36     !    - REAL(KIND(0.0D0)), i.e., double-precision real.
37     !
38     !    Therefore, the following should be noted.
39     !
40     !    - Other types of INTEGER/REAL may not be available on all platforms
41     !      (e.g., pgfortran 20 does not support REAL128).
42     !    - The standard does not specify the range of the default  integer. 
43     !    - The standard does not specify what the range and precision of the 
44     !      default real or the double-precision real, except that KIND(0.0D0)
45     !      should have a greater precision than KIND(0.0) --- no requirement 
46     !      about the range. 
47     !      
48     !    Consequently, the following should be observed in all Fortran code. 
49     !
50     !    - DO NOT use any kind parameter other than IK, IK_DFT, RP, RP_DFT,
51     !      SP, or DP unless you are sure that it is supported.
52     !    - DO NOT make any assumption about the range of INTEGER, REAL, or
53     !      REAL(0.0D0) unless you are sure. Be cautious about OVERFLOW! 
54     !      In particular, for integers working as the lower/upper limit of
55     !      arrays, overflow can lead to Segmentation Faults! 
56     !
